ARCHITECTURE CLARIFICATION (from Gemini audit + Em's question)
================================================================

HOW EDITING WORKS TODAY:
  Builder (tokenless)    → generates brand-new complete HTML → OVERWRITES everything
  Bulk scripts           → also regenerate complete HTML → OVERWRITES
  Claude with tokens     → reads file → targeted edit → safe
  Em directly on GitHub  → edits specific text → safe

STUDENT PROGRESS: stored in localStorage independently → always safe
STUDENT NOTES: stored in localStorage → always safe
CONTENT (words, slides, questions): inside HTML files → at risk from regeneration

Gemini suggested D1 database for content — assessed and rejected for content.
D1 is right for runtime teacher data (profiles, calendar).
Course content belongs in git (version controlled, AI-editable, free, offline-safe).

PLAN — PHASE 2: SEPARATE _data.js PER CHAPTER (new chapters only)
================================================================

Context:
  Currently template (HTML structure) and content (words, slides,
  questions) are in the same file. If the template needs updating,
  the content is at risk. Phase 2 separates them.

What to build:
  Builder generates TWO files instead of one:

  _data.js — only the content array:
    var WORDS = [
      {"word":"Accommodation","ipa":"/əˌkɒməˈdeɪʃən/",
       "def":"a place to stay","ex":"I booked <b>accommodation</b>..."},
    ];

  flashcards.html — template only, loads data externally:
    <script src="_data.js"></script>
    <!-- rest of engine uses WORDS from _data.js -->

  Builder UI changes:
  - Step 4 shows TWO download buttons: "Download template" + "Download data"
  - Placement path shows both files
  - Instruction: "Put both files in the same chapter folder"

  generateFlashcards(), generateSlides(), generateWorksheet() in
  builder.html: extracted data block becomes a separate returned string.

What this means in practice:
  - Fix a typo in a word → edit only _data.js → template untouched
  - Add a new game feature → edit only flashcards.html → content safe
  - Claude bulk update → only touches _data.js → never risks HTML
  - Builder regenerating template → _data.js stays, words are safe

Rollout: new chapters only. Existing 2,400 files stay as-is.
Migration of old chapters happens naturally when they need updates.

Files changed: builder.html (generators + Step 4 UI).

Verification:
  - Build any flashcard → two download buttons appear
  - Place both files in same folder → chapter loads correctly
  - Re-run builder on same chapter → only template changes, _data.js untouched
