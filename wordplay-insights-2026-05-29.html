The Week in Summary

What's working

You run a tight, verification-driven operation: you don't accept UI changes until Claude confirms them with Playwright across desktop, dark mode, and mobile, which catches real rendering bugs that would otherwise ship. Your bounded audit loops — "find and fix exactly 5 bugs then stop" or driving 169 pedagogy failures to zero — keep multi-file changes controlled and committed cleanly. You've also smartly used Claude as a verification layer to catch a teammate fabricating git commit hashes.

What's hindering you

Claude occasionally insisted work was already done before actually inspecting the rendered output (the amber confetti loop is the clearest example), and bulk fixes sometimes missed template variants and whitespace patterns, forcing follow-up passes. On your side, the biggest drag is structural: long Playwright audits repeatedly ran into session and weekly usage limits mid-task, while unstable MCP tooling added detours.

Quick wins

Turn your repeated workflows — audit loops, branch verification, version bumps — into Custom Skills so they're a single command. Pair that with Hooks to auto-verify and push commits at session end. Use Headless Mode to run your big Playwright audits from a script so they checkpoint and survive limit resets.

Suggestions

CLAUDE.md Additions

Add these to your project's CLAUDE.md to prevent the recurring friction above from repeating across sessions.

Design System (strict)

Multiple sessions had Claude introduce emojis, navy colors, or per-card colors, requiring polish passes to fix.

## Design Conventions (STRICT)
Accent colour: amber only — #E8A020 / var(--amber).
No teal, gold, navy, per-card colours. No emojis anywhere.
Only exceptions: ◐/◑ dark toggle, ◆ streak badge.
Dark mode must work on every element — use !important overrides.
Before every CSS edit: check these rules.

Template Variant Rule

Several sessions fixed only 1 of 3 game.html template variants and missed 254 Spanish pages with a different whitespace pattern.

## Multi-file Edits
When applying a fix to generated/templated files:
1. grep for ALL template variants and whitespace patterns first.
2. Confirm the file count per variant with me before running.
3. Never assume one pattern covers everything — check Spanish/vocab
   pages separately (they often differ in whitespace).

Verification Before Claiming Correctness

Claude repeatedly insisted confetti was already amber without re-checking, causing frustrating back-and-forths.

## Verification
Never claim a value or state is "already correct" from memory.
Re-read the actual source line and show a Playwright screenshot/
evaluate result proving the rendered output before concluding.
Always verify across: desktop, dark mode, mobile 360px.

Features to Try

Tools You're Under-Using

Custom Skills

Reusable single-command workflows

You repeatedly run the same multi-step workflows (Playwright lesson audits, branch verify+cherry-pick+merge, cache-bust version bumps across ~200 files). Skills make these one command and survive session limits.

Create .claude/skills/audit/SKILL.md:
"Run a resumable Playwright audit of all A1–C2 lessons,
log findings to audit-progress.json, fix missing
EXPLANATIONS/GAME_DATA, and report."
Then run /audit in any session.
Hooks

Auto-run at lifecycle events

A stop-hook already caught an unpushed commit. Hooks can auto-verify pushes and enforce the no-emoji/palette conventions without you having to remember to ask.

{
  "hooks": {
    "Stop": [{
      "hooks": [{
        "type": "command",
        "command": "git status --porcelain && git log origin/main..HEAD --oneline"
      }]
    }]
  }
}
Headless Mode

Run Claude non-interactively from scripts

Your Playwright audits kept getting cut off by session limits mid-task. Running batch audits headlessly in chunks avoids interactive-session limits and lets you resume cleanly.

claude -p "Run the lesson audit for A1-A2 only, fix missing
EXPLANATIONS, commit." --allowedTools "Edit,Read,Bash"
